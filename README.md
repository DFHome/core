# DFHome — веб-приложение для Умного дома Яндекса

Собственный веб-интерфейс для просмотра и управления устройствами Яндекс Умного
дома, а также создания/редактирования сценариев. Запускается одной командой
через `docker-compose`.

## Архитектура

- **backend** (FastAPI, Python) — прокси к двум API Яндекса:
  - официальный `api.iot.yandex.net` — устройства, комнаты, управление, запуск сценариев;
  - неофициальный `iot.quasar.yandex.ru` — создание/редактирование/удаление сценариев
    (официальный API этого не позволяет).
- **frontend** (React + Vite, за nginx) — SPA с дашбордом устройств, редактором сценариев
  и страницей настроек.

Токены хранятся в SQLite на docker volume и вводятся вручную через страницу «Настройки».

## Запуск

```bash
cp .env.example .env   # опционально — можно оставить пустым и ввести токены в UI
docker-compose up --build
```

Приложение будет доступно на **http://localhost:8080**.

## Получение доступа к аккаунту Яндекса

### 1. OAuth-токен (обязательно, для устройств/управления/запуска сценариев)

1. Зарегистрируйте приложение на https://oauth.yandex.ru/client/new/:
   - платформа: веб-сервисы;
   - права доступа (scope): `iot:view` и `iot:control`.
2. После создания приложения на странице приложения будет ссылка для получения
   отладочного токена (debug token) — либо пройдите стандартный OAuth-флоу:
   откройте `https://oauth.yandex.ru/authorize?response_type=token&client_id=<CLIENT_ID>`,
   авторизуйтесь и скопируйте `access_token` из URL, на который вас перенаправит браузер.
3. Вставьте токен в приложении на странице **Настройки → OAuth-токен**.

### 2. Cookie для сценариев (опционально, только для создания/редактирования)

Официальный API не поддерживает создание и редактирование сценариев, поэтому для этого
используется внутренний API, для которого нужна сессионная cookie:

1. Откройте https://yandex.ru/quasar/iot в браузере и войдите в свой аккаунт.
2. Откройте DevTools (F12) → вкладка **Network**.
3. Найдите любой запрос к `iot.quasar.yandex.ru` (например, при открытии списка сценариев).
4. Скопируйте значение заголовка `Cookie` целиком.
5. Вставьте его в приложении на странице **Настройки → Quasar Cookie**.
6. Если сервер отвечает 403, дополнительно скопируйте заголовок `x-csrf-token` из того же
   запроса и вставьте его в поле **Quasar CSRF-токен**.

Cookie периодически "протухает" — если создание/редактирование сценариев перестало
работать, обновите её тем же способом.

## Разработка без Docker

```bash
# backend (использует uv — https://docs.astral.sh/uv/)
cd backend
uv pip install --system -r requirements.txt
uvicorn app.main:app --reload

# frontend (использует pnpm)
cd frontend
pnpm install
pnpm run dev   # проксирует /api на localhost:8000, см. vite.config.ts
```

## Тесты

```bash
cd backend
uv run pytest
```

## Известные ограничения

- Внутренний (quasar) API Яндекса не документирован официально и может измениться в любой
  момент — это единственный способ создавать/редактировать сценарии программно. Вся
  quasar-специфичная логика изолирована в `backend/app/yandex/quasar.py`; поломка этого API
  не влияет на просмотр/управление устройствами и запуск существующих сценариев.
- При редактировании сценария, созданного не через это приложение, часть условий/действий
  может не распознаться (см. `parse_quasar_scenario` в `quasar.py`) — в этом случае
  приложение сохранит название сценария и предложит задать условия/действия заново.
- Приложение предназначено для личного использования в доверенной сети — токены хранятся
  в открытом виде в SQLite; не выставляйте порт 8080 в публичный интернет без дополнительной
  аутентификации/reverse-proxy.
